# AGENTS.md - AI Agent Context for Telnyx Web Client

## Project Overview

**Project Name:** Telnyx Web Client
**Purpose:** Webapp to send and receive messages from Telnyx
**Architecture:** Full-stack monorepo with Rust backend and React frontend

## Tech Stack

### Backend (Rust) - Workspace Architecture
- **Language:** Rust (Edition 2024)
- **Workspace:** Cargo workspace with resolver v3
- **Web Framework:** Actix-web 4.x with actix-inertia 0.1.0
- **Async Runtime:** Tokio 1.49.0
- **Security:** actix-web with rustls and secure-cookies
- **CORS:** actix-cors 0.7.1
- **Rate Limiting:** actix-governor 0.10.0
- **File Handling:** actix-multipart 0.7.2, actix-files 0.6.9, actix-embed 0.1.0
- **Serialization:** serde 1.0.228, serde_json
- **Logging:** tracing 0.1.43 with color-eyre
- **Config:** dotenvy 0.15.7
- **Dependency Management:** cargo-hakari for centralized workspace dependencies
- **Architecture:** Domain-driven design (DDD) with layered crates

### Frontend (React + TypeScript)
- **Framework:** React 19.2.0 + TypeScript 5.9.3
- **Build Tool:** Vite (rolldown-vite 7.2.5)
- **Navigation:** Inertia.js 2.3.6
- **UI Library:** shadcn/ui (New York style, neutral base)
- **State Management:** React State + Inertia.js
- **Styling:** Tailwind CSS v4.1.18
- **Forms:** react-hook-form 7.71.1 + zod 4.3.5
- **Icons:** lucide-react 0.562.0

## Backend Patterns (Rust)

### Workspace Architecture
- **DDD (Domain-Driven Design):** Clear separation of concerns across layers
- **Dependency Flow:** bin/web → web → application → domain → infrastructure
- **Binary Name:** `web-server` (changed from `server`)
- **Entry Point:** `bin/web/src/main.rs`

### Server Setup (crates/web/src/server.rs)
- Uses Actix-web with 5 workers
- Binds to config address (default: 127.0.0.1:8080)
- Graceful shutdown on SIGINT/SIGTERM
- Inertia.js integration via actix-inertia
- Middleware: NormalizePath, Compress, Logger, CORS, Governor (rate limiting)

### Configuration (crates/infrastructure/src/config/)
- Loaded from environment variables using WebConfig
- Supports dotenv for .env file loading
- Uses `bon` crate for builder patterns
- Custom error types via `thiserror`

### Handlers (crates/web/src/handlers/)
- **auth.rs**: login, signup, forgot_password, reset_password
- **inertia.rs**: version endpoint for Inertia asset versioning
- All handlers check `x-inertia` header for JSON vs HTML responses

### Inertia.js Integration (crates/web/src/inertia.rs)
- Development mode: Generates HTML shell with Vite HMR
- Production mode: Serves pre-built `dist/index.html`
- Supports React Fast Refresh in dev mode
- Replaces `{{DATA_PAGE}}` placeholder with serialized props

### Logging
- Uses tracing crate with structured logging
- color-eyre for better error reporting
- Filter: `info,<crate_name>=debug,tokio=info`

### Form Validation Pattern
```tsx
import { useForm } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import * as z from "zod";
import {
  Form,
  FormControl,
  FormField,
  FormItem,
  FormLabel,
  FormMessage,
} from "@/components/ui/form";
import { Input } from "@/components/ui/input";

const formSchema = z.object({
  email: z.string().email("Invalid email"),
  password: z.string().min(8, "Password too short"),
});

function YourForm() {
  const form = useForm<z.infer<typeof formSchema>>({
    resolver: zodResolver(formSchema),
    defaultValues: { email: "", password: "" },
  });

  function onSubmit(values: z.infer<typeof formSchema>) {
    console.log(values);
  }

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)}>
        <FormField
          control={form.control}
          name="email"
          render={({ field }) => (
            <FormItem>
              <FormLabel>Email</FormLabel>
              <FormControl>
                <Input type="email" {...field} />
              </FormControl>
              <FormMessage />
            </FormItem>
          )}
        />
      </form>
    </Form>
  );
}
```

## Key Considerations for AI Agents

### Backend (Rust) Work:
1. **Workspace Structure:** Code is split across crates in `crates/` and `bin/`
2. **DD Layers:** Respect the layered architecture (web → application → domain → infrastructure)
3. **Binary Entry:** Main entry point is `bin/web/src/main.rs`
4. **Binary Name:** Use `web-server`, not `server`
5. **Actix-web:** Framework with async/await
6. **Inertia.js:** Integration for server-side rendering
7. **Rust Conventions:** snake_case, proper error handling with Result<T, eyre::Error>
8. **Logging:** Use tracing macros (info!, debug!, error!)
9. **Dependencies:** All crates depend on `workspace-hack` for shared deps
10. **Configuration:** Use `infrastructure::config::WebConfig` for environment variables

### Which Crate to Modify?

| Task | Crate Location |
|------|----------------|
| HTTP routes, handlers, server setup | `crates/web/` |
| Request/response handling | `crates/web/src/handlers/` |
| Inertia.js integration | `crates/web/src/inertia.rs` |
| Configuration, env vars | `crates/infrastructure/src/config/` |
| Business logic, entities | `crates/domain/` |
| Use cases, orchestration | `crates/application/` |
| Binary entry point, main | `bin/web/` |

### Frontend (React) Work:
1. Files are in `web/src/` directory
2. Pages go in `web/src/Pages/`
3. Use shadcn/ui components from `@/components/ui/*`
4. Form validation: react-hook-form + zod
5. Icons: lucide-react
6. Follow "new-york" style and neutral color scheme
7. No `as any` or `@ts-ignore` - maintain type safety

### When Modifying UI:
1. VISUAL changes (styling, layout, animation) → Delegate to frontend-ui-ux-engineer agent
2. LOGIC changes (state, API calls, business logic) → Can handle directly

### Backend Integration:
- Frontend communicates via Inertia.js, not direct fetch
- API proxy: `/inertia` routes to backend at `http://127.0.0.1:8080`
- Use Inertia.js visit method for programmatic navigation
- All backend routes are defined in `crates/web/src/server.rs`

## Notes

- **Monorepo**: Both backend (Rust workspace) and frontend (React/Vite) in one repository
- **Inertia.js**: Provides SPA-like experience without client-side routing complexity
- **Actix-web**: High-performance Rust web framework
- **shadcn/ui**: Full suite of accessible, customizable React components
- **Modern stack**: Rust 2024 edition, React 19, TypeScript 5.9, Tailwind CSS v4
- **Strict typing**: Both Rust and TypeScript with strict mode enabled
- **Security**: TLS support, secure cookies, CORS configuration, rate limiting
- **Binary naming**: The production binary is named `web-server`, not `server`
